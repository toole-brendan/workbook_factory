# Sheet Guide — Distributed-Shipbuilding workbooks (shared, both programs)

Reference for hand-building sheet modules: how to build one, the style rules, the
locked sheet skeleton, and the cross-sheet conventions. Copy-from builders
(section block, input table, cross-sheet link, FY series, native table, QA row)
live in `workbook_core/sheet_snippets.md`; the deeper raw SpreadsheetML mechanics
(cell encoding, styles.xml, tables, defined names, data validation, conditional
formatting) live in `workbook_core/ooxml_cheat_sheet_xlsx.md`.

---

## Building a sheet

A sheet module is self-contained: it imports the builders from `workbook_core`
(`primitives` / `styles` / `tables`), composes rows, and returns a
`WorksheetSpec`. The packager (`workbook_core.lib.package_workbook`) derives the
tab name, wires any native tables / defined names, and zips the workbook.

1. Copy `sheet_base_template.py` to `workbook_{sub,ddg}/sheets/<sheet_name>.py` —
   it has the metadata block, the title banner, and the BUILD-HERE marker.
2. Write the one-paragraph INTENT and the LAYOUT row-map in the docstring.
3. Set the metadata: leave `TAB_NAME = None` to derive the tab from the filename
   (`pop_coefficients.py` → "Pop Coefficients"), or set it explicitly when you
   want a name the filename can't express (`"SCN Annual"`, `"Prime 10-K"`). Set
   `SHEET_GROUP` (the tab color + tab-block order derive from it — see Groups &
   tab colors below), and set `COLS`.
4. Build `_build_rows()` at the marker — `banner_row` / `write_row` /
   `build_table`, placing content from column B (`start_col=1`) and row 2. Paste
   builders from `sheet_snippets.md` as file-private helpers.
5. If the sheet exposes load-bearing cells, add accessor functions returning
   absolute refs (see `sheet_snippets.md` → cross-sheet link).
6. Add the module to `workbook_{sub,ddg}/sheets/__init__.py`'s `SHEETS` list, in
   tab order. The registry is a list of modules — there is no count to bump and
   no name to register; the module owns its name.

**Inspecting.** Emitted-cell truth is auto-generated, never hand-maintained. Read
`workbook_{sub,ddg}/reports/sheet_probe/<sheet_name>.{md,json}` for current cells
(refs, styles resolved to their `S_*` name, values, formulas, banners, tables,
defined names) — parsed from the emitted XML. Regenerate via
`workbook_{sub,ddg}/tools/sheet_probe.py`.

## Human workbook standard

The generated workbook should look like one a human analyst built by hand in Excel —
not like a workbook that explains it was generated. Do not emit "AI workbook"
scaffolding unless the user explicitly asks for it.

Avoid generated-workbook artifacts:
- `Guide`, `README`, `Instructions`, `How to Use`, or `Workbook Overview` tabs, or
  any tab whose only job is to explain the model. Groups are ordering/color
  metadata, not a checklist — don't add a tab just because a group exists (the
  `guide` group is optional and usually unused).
- visible prose such as "This workbook…", "This sheet…", "Use this tab to…", "The
  purpose of this sheet is…", or "Generated by…".
- long narrative title banners, section headers, or sub-section headers.
- visible "role" labels in cells (e.g. `Role: Input Surface`, `Model Engine`). The
  `S_*` styles already encode cell *purpose*; keep purpose internal, never as text.

Keep authoring notes in the source, not in cells. The module docstring carries
INTENT, the LAYOUT row-map, and any agent-facing explanation; none of that belongs
in a worksheet cell.

Visible workbook text is compact and analyst-native:
- title banner: the tab name, not a sentence;
- section banners: a `§N - short noun phrase` label (see Section numbering), not a
  full sentence, no trailing period;
- sub-section banners: only when they break up a genuinely dense block;
- notes/context: only where a human workbook would carry them (Sources is the one
  standing exception, a one-line context note per citation).

## Tab names

The pipeline does not mandate any specific tab names — it mandates only the group
taxonomy (color + order, see Groups & tab colors). Name tabs the way an analyst
would.

**Minimize underscores in visible tab names.** An underscore reads as machine output
on a worksheet tab; write `Control Panel`, `TAM Model`, `Prime 10-K` — not
`Control_Panel` / `TAM_Model`. Underscores stay correct everywhere they are
*identifiers* rather than visible labels: Python module names (`control_panel.py`),
accessor functions, native table names (`tbl_subawards`), and defined names
(`portfolio_tam`). The discouragement is about the visible tab only.

Leave `TAB_NAME = None` when the filename already expresses the tab — the default
derivation drops the filename's underscores (`pop_coefficients.py` → `Pop
Coefficients`), so they never reach the tab. Set `TAB_NAME` explicitly only when the
visible tab needs punctuation, an acronym, or wording the filename can't carry
(`SCN Annual`, `Prime 10-K`). Don't use the group keys (`inputs`, `data`, `model`,
…) as tab names — they are internal metadata.

## The sheet skeleton

Every sheet shares the same chrome; only the section content varies.

- **Gutter (column A).** A narrow ~1.5-char gutter. No content lives there; banner
  fills do not extend into it. Content starts at column B — pass `start_col=1` to
  `write_row` / `build_table`. The `COLS` widths you pass to `worksheet()` are the
  content columns only; `worksheet(with_gutter=True)` prepends the gutter.
- **Row 1 — auto-injected blank.** `worksheet(with_gutter=True)` emits an empty
  `<row r="1"/>` so every other row shifts down by 1. **Do not emit row 1.**
- **Row 2 — title banner** (`S_TITLE_SHEET`, black bg / white bold), full content
  width. The template derives the text from `TAB_NAME`/filename — keep it that way
  so the banner and the tab never drift.
- **Row 4+ — section banners + body.** Section banners use `S_TITLE_SECTION`
  (mid-gray), sub-sections `S_TITLE_SUBSECTION` (light-gray). See **Row spacing
  rhythm** below for the exact blank-row pattern.

Locked workbook standards (enforced in `worksheet()` — you cannot set them per
sheet): no gridlines, 10pt row height, no frozen panes, no merged cells.

### Row spacing rhythm

Compact, like a hand-built model — not airy with decorative blank rows.

- **1 empty row** after a section banner, before its first sub-section banner,
  table header, or content row.
- **1 empty row** after a sub-section banner, before that sub-section's header or
  content row.
- **2 empty rows** between the last content row of a section and the next section
  banner.
- Sections do not need sub-sections — use a sub-section banner only to break up a
  dense section, never as decorative structure.
- Don't add extra spacer rows to make the sheet look fuller or more "explained."

### Section numbering

Prefix every section banner with a `§` number, and sub-sections with a letter
suffix on the parent number:

- **Section banner — `§N - Title`**: the section sign `§`, a sequential integer
  (per sheet, from §1), ` - `, then a short noun phrase. E.g. `§1 - Marauder`,
  `§2 - Annual O&S and lifecycle`.
- **Sub-section banner — `§Na - Title`**: the parent section number + a lowercase
  letter. E.g. `§6a - Total per-ship-per-year`, `§6b - Per-element vs antecedent`.

The number is the payoff: it gives each block a short, stable ID you can point at
from anywhere — a source-note cell (`MSAR Dec 2023 §4`), a cross-sheet reference
(`Calcs §1 row 11`), or a checks tie-out (`O&S §3 vs §4`) — without quoting a row
number that moves. Keep the title itself short; the numbering carries the structure
so the words don't have to. Use a plain hyphen in ` - ` (not an en/em dash) so the
dash-normalization switch leaves it alone.

## Column widths

`COLS` are the content columns only (column B onward); `worksheet(with_gutter=True)`
prepends the ~1.5-char gutter. Size widths to the content, the way an analyst would
— not padded wide:

- first text / label column: ~36–44;
- numeric / FY columns: ~12–14;
- short category / status columns: ~10–14;
- one-line context / source-note columns: ~24–32, used sparingly;
- avoid very wide columns as a substitute for a shorter label.

No wrap, no merges. If a label doesn't fit, shorten it, split the block, or move the
detail into the source docstring — never widen a column to fit a sentence.

## Style rules (every sheet, both workbooks)

**Data flow**
- Editable values cluster on the inputs surface (the `inputs` group). Downstream
  sheets *link* to them via accessor functions, never by re-typing the value or
  hardcoding a row number. A canonical accessor returns an absolute ref string
  (e.g. `"'Control Panel'!$C$15"`); use `sheet_ref` / `abs_ref` / `qsheet` to
  build it (see snippets → formula & ref helpers).
- Sheets that compute load-bearing values expose an accessor (e.g.
  `bc_supplier_coeff_cell()`) so downstream sheets link to the *rendered* cell,
  not a recomputed copy. A link that bypasses the calc and recomputes can pass
  while the calc is broken.
- Every headline number is formula-driven. Narrative prose is self-referential so
  it stays accurate when inputs flex.

**Cell font colors** (numeric cells only — text labels and round-integer metadata
stay default black):
- **Blue** (`S_NUM_INPUT` / `S_PCT_INPUT`) — a hardcoded numeric input.
- **Black** (`S_NUM` / `S_PCT`) — a derived value (formula result).
- **Green** (`S_LINK_NUM` / `S_LINK_PCT`) — a pure cross-sheet link
  (`=Sheet!Cell`). Apply it explicitly; there is no auto-detection — the style you
  pass is the style rendered.

**Number formats** (universal; set by the `S_*` style, never a per-cell format):
- Numbers: comma + 1 decimal (`#,##0.0`); a zero renders as a dash `-`.
- Percentages: italic, 1 decimal (`0.0%`); zero renders as a dash.
- Dollar units (`$M`, `$M per year`, `$/unit`) live in the section banner text,
  never as a cell format.
- **Exception — round-integer metadata.** Ship counts, page numbers, fiscal/base
  years, line-item IDs use `S_DEFAULT` (no decimal, no color) — they are labels,
  not analytical numbers. Judgment call: if a value carries a meaningful decimal
  in this dataset (service life 38.9 yr, ratio 1.42x), use `S_NUM`.

**No merges · no text wrap · no notes/quality columns** (Sources is the sole
exception, carrying a one-line context note per citation). If a label doesn't fit,
restructure or shorten — never widen the row or wrap.

**Native Excel Notes (hover cards)** — the rule above bans visible note/commentary
*columns*; a native Excel Note (the old red-triangle / yellow hover card) is
different and allowed, sparingly. Anchor it on the **cell it explains — the
headline value / the number itself**, never on the row's label cell and never on a
separate note/commentary cell (there is no note column to hang it on). Keep it
short and analyst-voiced: one tight line of derivation or caveat, the way a human
modeler annotates a figure — not a generated-sounding sentence. Declare notes on
the returned `WorksheetSpec` — `WorksheetSpec(ws, notes=[ExcelNote("H8", "…")])` —
and the packager wires the parts (see snippets → native Excel note). Notes are
presentation metadata, **not** the audit trail: ref IDs, evidence grades, source
records, and checks still belong in real tables / Sources / Validation, never
hidden in a Note.

**Explicit styles** — every style argument is required (no defaults):
`cell(style=)`, `banner_row(style=)`, `write_row(styles=)` (int broadcast or a
list whose length equals values), `build_table(header_style=, col_styles=)`. For a
plain text label, pass `S_DEFAULT` explicitly.

**Total / subtotal rows** — use `total_row(r, values, *, styles, n_cols)`, never a
hand-built `write_row`. The medium top divider draws per-cell, so a bare `S_BOLD`
label or `S_DEFAULT` spacer breaks the line; `total_row()` upgrades each style to
its bordered variant and pads to `n_cols` so the rule runs unbroken across the
block (gutter excluded). Pass the plain base styles + the block width — see
snippets → totals row.

**Typography** — Arial 8pt everywhere (locked in `styles.py`).

**Indentation & row hierarchy** — indent a label only to show it is a *component
of* the line item or subtotal it sits under (parent → children), never as
decoration or as a stand-in for a section/sub-section banner.
- Indent the *label column only* of a styled-range model block (cost build-ups,
  roll-forwards, P&L-like blocks, TAM→SAM bridges, reconciliations). Use
  `S_LABEL_INDENT_1` for a direct component, `S_LABEL_INDENT_2` for a rare nested
  one; if a block wants a third level, split the section instead.
- Parent lines, standalone inputs, and subtotal/total rows stay flush-left
  (`S_BOLD` / `S_TOTAL`) — the total reads cleanest un-indented under its
  components.
- Indent components, not peers. A scenario table (`Base` / `Upside` / `Downside`)
  is flush-left records, not a parent→child block.
- This is *visual* indentation only and is independent of `outline_level` (the
  collapsible-group feature, see snippets → collapsible group); a row can be
  indented, outlined, both, or neither.
- Do **not** indent rows in native tables, source lists, output contracts, or
  lookup tables — those are flat records. If rows there need hierarchy, the block
  should be a styled range, not a ListObject.
- Do **not** fake indentation with leading spaces in the label text (`cell()`
  preserves them); that hides structure inside a value. Use the indent style.

Numeric / FY column headers may use `S_HEADER_CENTER` (centered over the column);
text-label headers stay `S_HEADER_LEFT`. See snippets → model hierarchy block,
FY-columns series.

### Style cheat sheet (pick by cell purpose)
```
Text label, ID, scenario tag        S_DEFAULT
Indented component label            S_LABEL_INDENT_1   (nested: S_LABEL_INDENT_2)
Round-integer metadata              S_DEFAULT      (count, page, fiscal/base year)
Column header row (text)            S_HEADER_LEFT
Column header (numeric / FY)        S_HEADER_CENTER
Bold black label                    S_BOLD
Total / subtotal divider (text)     S_TOTAL

Number, hardcoded input             S_NUM_INPUT
Number, derived (formula)           S_NUM
Number, cross-sheet link            S_LINK_NUM
Number, total row (derived)         S_NUM_TOTAL
Number, total row (hardcoded)       S_NUM_INPUT_TOTAL

Percent, hardcoded input            S_PCT_INPUT
Percent, derived                    S_PCT
Percent, cross-sheet link           S_LINK_PCT
Percent, total row                  S_PCT_TOTAL

Title banner (row 2)                S_TITLE_SHEET
Section banner                      S_TITLE_SECTION
Sub-section banner                  S_TITLE_SUBSECTION
```

## Groups & tab colors

Every sheet belongs to one logical **group** (its chapter in the workbook). The
group drives both the **tab color** (one color per group) and the **tab order**
(each group is one contiguous block, in `groups.SHEET_GROUPS` order). Set
`SHEET_GROUP` to one of the keys below and let `TAB_COLOR = group_color(SHEET_GROUP)`
— don't pass arbitrary hex. `package_workbook()` asserts the blocks stay contiguous
and in order, so a mis-tagged or misplaced tab fails the build.

```
guide        teal       optional/rare: only if a human workbook here would carry one
inputs       ochre      the editable knobs (the only place you change numbers)
data         burgundy   extracted source evidence / data dumps
model        indigo     the TAM -> SAM calc engine
outputs      green      the deck-facing figure contract
validation   gray       audit, checks, sensitivity, exclusion trails
sources      navy       provenance / citations
chartdata    charcoal   the deck chart-data loader tab (machine-readable; sorts last)
```

Groups are ordering + color metadata, not a checklist of tabs: declare only the
groups a workbook needs and skip the rest (no group is required to appear). In
particular don't add a `guide`/readme tab just because the group exists — see
**Human workbook standard**. The canonical order + the exact hexes live in
`workbook_core/groups.py` (the single source of truth); reorder the blocks or
retune a color there.

## Tables & defined names

- **Styled range (`build_table`)** — the default for model blocks: a header row +
  styled data rows, returns `(rows_xml, next_row)`. No filter, no table name.
- **Native Excel table (`ExcelTable`)** — a real ListObject (structured refs,
  filter persistence). The cell data still lives in the worksheet; the table is a
  separate package part the packager wires when you attach an `ExcelTable` to the
  returned `WorksheetSpec`. Use it for filterable / structured ranges: research
  dumps, deck-facing contracts, source lists, scenario / lookup tables. Don't wrap
  every calc block in one.
- **Defined names** — attach `defined_names={name: ref}` to the `WorksheetSpec` to
  publish a stable workbook-scoped name (e.g. `portfolio_tam` →
  `'TAM Model'!$H$8`). Names follow Excel's rules — `[A-Za-z_][A-Za-z0-9_.]*`, not
  a cell-reference shape, not a reserved `_xlnm` name (the packager validates this
  at build). See the cheat sheet "Defined names" (and the `localSheetId` trap =
  zero-based sheet *position*, not `sheetId`) before adding sheet-scoped names.

### Native table visual discipline

A native table is for ListObject behavior — filters, structured references, and a
stable flat-record contract — not for visual decoration.

`ExcelTable` defaults to workbook_core's **named no-format table style**
(`NO_FORMAT_TABLE_STYLE`). That style is defined in `styles.xml`, points every
table-style element at an empty differential format, and draws no fill, border,
font, stripe, header rule, or final border. The visible look comes from the cells
you already wrote with `S_*` styles: header underline via `S_HEADER_LEFT`, ordinary
borderless body rows, and total dividers only where you use `total_row()`. A *named*
style (rather than no `<tableStyleInfo>` or the old `name="None"`) is the
round-trip-safe choice — Excel can inject its own banded default on open/save when a
native table has no durable named style to preserve.

Use native tables for flat, filterable / structured ranges:
- extracted data / evidence dumps;
- source and citation inventories;
- deck-facing data contracts;
- scenario, lookup, mapping, and classification tables.

Do **not** use a native table for ordinary model blocks, hierarchy blocks, TAM/SAM
bridges, P&L-style sections, roll-forwards, subtotal stacks, or any block where row
position and section structure matter more than ListObject behavior — use
`build_table()` for those. If you only need filter dropdowns with full visual
control, use a styled range + `worksheet(auto_filter=...)`.

Do **not** pass `style=None` merely to remove formatting — the default named
no-format style is the round-trip-safe way to do that. Reserve `style=None`
(emits no `<tableStyleInfo>` at all) for low-level OOXML cases where the absence is
intentional. Pass a built-in style such as `TableStyleMedium2` only when a data dump
genuinely reads better banded.

## Direct XML audit (formula grep)

In **source**, formulas are written as strings beginning with `=` (e.g.
`"=SUM(B2:B5)"`). In **emitted OOXML** the leading `=` is stripped, so the cell
carries `<f>SUM(B2:B5)</f>` — grep for `<f>`, not `<f>=`.

The build emits canonical, full-text formulas. After Excel opens and re-saves the
file, repeated formula runs can convert to OOXML shared-formula form: the first
cell carries `<f t="shared" ref="C6:H6" si="0">…</f>` and dependents carry
`<f t="shared" si="0"/>` with no inline text. Both forms are valid and evaluate
identically; only a raw `grep <f>…</f>` misses the dependents. **For a formula
audit, rebuild from source first** (`python build_workbook.py`) and inspect the
freshly built file, or use a shared-formula-aware reader — not raw regex on an
Excel-resaved file.

## Placeholder convention (value not yet researched)

- **Numeric placeholder** — enter the working estimate with `S_NUM_INPUT` /
  `S_PCT_INPUT` (blue) and tag it "TBD" or "Pending <source>" in a note column.
  Blue font is the visual cue that a numeric cell is editable / not yet final.
- **Text placeholder** — enter the descriptor with `S_DEFAULT`; the note tag does
  the work (text cells carry no color cue).
- **Sources** — leave the ref id with a `PEND-` prefix until the citation lands.
